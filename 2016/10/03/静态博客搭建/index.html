    静态博客搭建

    一、操作过程

    准备工作

    1. git for windows
    2. node.js
    3. 注册github账号

    对git的部署

    首先将git添加到环境变量中

    将注册的邮箱和用户名上传

    $git config--global user.name "Your name"

    $git config--global user.name "email@example.com"

    利用如下语句生成秘钥和公钥，然后打开公钥文件将其中的内容全部复制到github账号的ssh key中

    $ssh-keygen -t rsa -C youremail@example.com

    检查是否可以连接

    ssh git@github.com

    确定连接成功后执行克隆远程仓库和提交本地仓库的操作

    HEXO

    首先安装hexo

    npm install hexo -g

    新建一个目录blog

    mkdir blog

    转移到目录blog下，并对hexo进行初始化

    cd bolg
    hexo init

    启动hexo server

    hexo server

    在浏览器中访问localhost:4000
    如果能够成功访问则说明hexo配制成功

    将本地hexo与远程github关联

    修改配置文件

    在blog中找到配置文件_config.yml,用记事本或其他编辑器打开并对其进行如下修改

    deploy:
    type: git
    repository: git@github.com:yruri/yruri.github.io.git
    branch: master

    安装hexo-deployer-git

    npm install hexo-deployer-git -S

    生成静态网页

    hexo g

    部署到远程github

    hexo d

    (hexo g和hexo d是hexo generate 与hexo deploy的缩写)

    设置主题

    安装

    $ git clone git@github.com:forsigner/fexo.git themes/fexo

    启用

    修改你的博客根目录下的config.yml配置文件中的theme属性，将其修改为fexo。

    更新

    $ cd themes/fexo
    $ git pull
    更新前，最好先将themes/fexo/_config.yml文件备份

    (ps:该主题模版没有找到配置手册，还有许多功能正在摸索中配置)

    二、学习总结

    在本次学习中首先是因为别的事情耽搁了一段时间，十月二日才开始搭建，在三号晚上完成了。在这个过程中还是遇见了一些问题，但是通过百度都比较轻松的解决了。
    比如说：

    1. 不小心改动了“id_rsa.pub”文件，导致和远程github上存储的ssh不一致。
    2. 在第一次进行hexo d操作时，不小心中止了操作，结果使得“.deploy_git”文件夹里的文件不完整，第二次hexo d的时候报错，这时候将原来的“.deploy_git”文件夹删除，在重新操作就行了。
    3. 在markdown的语法中标题应该前后都加上#号，虽然在看操作手册的时候标题后的#号是可有可有无的，但是如果不加上的话在博客上前面的#会显示出来，可能是因为兼容性，但是还没有找到具体原因。

    虽然因为动手搭建的时间比较晚没有参与到群邮件的讨论，但是在完整的看了一遍后还是发现了很多自己在操作中没有注意或者说在意的问题，也避免了很多错误。

    最后：我的博客完成了。
